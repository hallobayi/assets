📦 Pilihan CDN untuk Assets

Dokumentasi lengkap tentang Content Delivery Network (CDN) yang tersedia untuk project Halo Bayi, termasuk informasi dukungan Content Security Policy (CSP).

1. jsDelivr ✓ CSP SUPPORT

Status: MENDUKUNG Content Security Policy

Rekomendasi: Primary CDN untuk production

Keunggulan:

  • Kompatibel penuh dengan strict CSP headers
  • Sistem cache optimal dengan versioning
  • Fallback otomatis ke multiple mirrors
  • CORS headers yang tepat
  • Support Subresource Integrity (SRI)

URL Pattern:

https://cdn.jsdelivr.net/gh/hallobayi/assets@1.1.27/vendors/jquery/jquery.min.js?r=1782870997

Contoh CSP Header:

Content-Security-Policy: script-src 'self' https://cdn.jsdelivr.net

2. GitHub Pages ✗ NOT SUPPORT CSP

Status: TIDAK mendukung Content Security Policy

Rekomendasi: Development/testing only

Keterbatasan:

  • Akan di-block oleh strict CSP policies
  • Tidak ada integrity hash support
  • CORS headers terbatas
  • Tidak cocok untuk production dengan security ketat

URL Pattern:

https://hallobayi.github.io/assets/vendors/jquery/jquery.min.js

Use Case: Development lokal, testing tanpa CSP, prototyping

3. StaticDelivr ✗ NOT SUPPORT CSP

Status: TIDAK mendukung Content Security Policy

Rekomendasi: Fallback untuk non-CSP environments

Keterbatasan:

  • Similar dengan GitHub Pages - blocked by CSP
  • Tidak support strict-dynamic directive
  • Kurang optimal untuk modern security policies

URL Pattern:

https://cdn.staticdelivr.com/gh/hallobayi/assets/1.1.27/vendors/jquery/jquery.min.js

Use Case: Alternative CDN untuk environment tanpa CSP requirements

💡 Rekomendasi Implementasi:
  • Production Environment: Gunakan jsDelivr sebagai primary CDN. Pastikan CSP headers di-set dengan benar di server.
  • Development Environment: Bisa menggunakan GitHub Pages atau StaticDelivr untuk kemudahan testing.
  • Staging Environment: Gunakan jsDelivr untuk simulasi kondisi production yang akurat.

⚙️ Contoh Implementasi

Cara 1: Pilih CDN Sesuai Environment (Recommended)

<!-- Production: CSP-Compatible --> <script src="https://cdn.jsdelivr.net/gh/hallobayi/assets@1.1.27/vendors/jquery/jquery.min.js?r=1782870997"></script> <!-- Development: Non-CSP --> <!-- <script src="https://hallobayi.github.io/assets/vendors/jquery/jquery.min.js"></script> -->

Cara 2: Fallback Chain

<script src="https://cdn.jsdelivr.net/gh/hallobayi/assets@1.1.27/vendors/jquery/jquery.min.js?r=1782870997"></script> <script> // Fallback jika primary CDN gagal if (typeof jQuery === 'undefined') { document.write('<script src="https://hallobayi.github.io/assets/vendors/jquery/jquery.min.js"><\/script>'); } </script>

📝 Notes

  • Version saat ini: 1.1.27
  • Parameter ?r=1782870997 pada jsDelivr adalah cache buster untuk force refresh
  • Selalu test CSP compatibility di staging sebelum deploy ke production
  • Untuk assets custom lainnya, gunakan pattern URL yang sama sesuai CDN pilihan
